# 选择性生图功能修复计划

## 问题分析

### 根本原因
用户发送"生成第5张图"时没有得到响应，问题出在以下环节：

1. **消息属性不匹配**：`getLastAmazonPlan()` 检查 `msg.isAmazonPlan` 属性，但在处理消息时可能没有正确设置这个属性
2. **解析函数不一致**：代码中有两个解析函数 `parseAmazonVisualPlan` 和 `parseAmazonPlan`，可能造成混淆
3. **缺少调试日志**：没有足够的日志来追踪问题所在

### 当前代码流程
1. 用户发送图片 → 亚马逊专家返回方案 → 设置 `isAmazonPlan: true` 和 `planImages`
2. 用户发送"生成第5张图" → 检测到选择性生图请求 → 调用 `getLastAmazonPlan()`
3. 如果找到方案 → 生成选中的图片

## 修复方案

### 方案一：添加详细调试日志
在关键位置添加日志，追踪选择性生图请求的处理流程

### 方案二：确保属性正确设置
确保 `isAmazonPlan` 和 `planImages` 属性在收到亚马逊方案时正确设置

### 方案三：简化解析逻辑
统一使用一个解析函数，避免混淆

## 修改文件

### `src/app/page.tsx`
- 在 `getLastAmazonPlan` 函数中添加日志
- 在选择性生图处理逻辑中添加日志
- 确保 `isAmazonPlan` 属性正确设置

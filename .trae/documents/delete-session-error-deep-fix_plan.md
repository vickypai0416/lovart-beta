# 关闭对话任务时 React 错误深度修复计划

## 问题分析

### 根本原因
React 错误 #185 是因为在渲染过程中更新了状态。从错误堆栈来看，问题出现在 `Array.map` 操作中。

可能的问题点：
1. **闭包问题**：在 `setTimeout` 中访问的 `currentSessionId` 和 `sessions` 是闭包捕获的值，可能不是最新的
2. **条件渲染问题**：在渲染过程中访问 `message.planImages` 可能导致问题
3. **状态更新冲突**：删除会话时的状态更新可能与其他状态更新冲突

### 当前代码位置
- `src/app/page.tsx` 第227-253行的 `handleDeleteSession` 函数
- 消息列表渲染代码（第1540-1678行）

## 修复方案

### 方案一：使用 useRef 避免闭包问题
使用 `useRef` 存储最新的状态值，避免闭包捕获过期值

### 方案二：简化删除会话逻辑
将删除会话逻辑简化，减少状态更新次数

### 方案三：添加条件检查
在渲染过程中添加更严格的条件检查，确保 planImages 存在

## 修改文件

### `src/app/page.tsx`

## 实施步骤

1. 添加 useRef 来存储最新的状态值
2. 修改 handleDeleteSession 函数，使用 ref 中的最新值
3. 在渲染过程中添加条件检查，确保 planImages 存在
4. 添加错误边界组件来捕获渲染错误

## 预期效果

修复后，关闭对话任务时不会再出现 React 错误 #185

# 选择性生图模式匹配修复计划

## 问题分析

用户报告说"生成第5张图时"没有得到请求。当前的选择性生图检测模式无法匹配这种带有"时"字结尾的表达。

### 当前模式列表
```javascript
const patterns = [
  /生成第(\d+)张图/,
  /生成第(\d+)张/,
  /生成图(\d+)/,
  /只生成第(\d+)张/,
  /只要第(\d+)张/,
  /请生成第(\d+)张/,
  /生成第(\d+)张图片/,
];
```

### 问题
用户输入"生成第5张图时"，由于后面有"时"字，正则表达式 `/生成第(\d+)张图/` 无法匹配。

## 修复方案

### 方案一：修改正则表达式
使用更宽松的正则表达式，允许在"张图"后面有任意字符

### 方案二：添加更多模式
添加更多匹配模式来覆盖不同的表达方式

### 方案三：使用更灵活的匹配逻辑
使用更灵活的匹配逻辑来提取数字

## 修改文件

### `src/app/page.tsx`
修改 `isSelectiveGenerationRequest` 函数，添加更多匹配模式

## 实施步骤

1. 修改正则表达式，使其能够匹配"生成第5张图时"这样的表达
2. 添加更多模式来覆盖不同的表达方式
3. 测试新的匹配逻辑

## 预期效果

修复后，用户可以使用以下表达方式来选择生成特定图片：
- "生成第5张图"
- "生成第5张图时"
- "帮我生成第3张"
- "生成第2和第4张图"

等等

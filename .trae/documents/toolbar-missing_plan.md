# 画布工具栏消失问题排查与修复计划

## 问题分析

用户反馈画布的工具栏消失了。从代码分析来看，工具栏在 `InfiniteCanvas.tsx` 第672行定义：

```tsx
<div className="flex-shrink-0 border-b bg-background px-4 py-2">
```

工具栏是画布组件的顶部子元素，使用 `flex-shrink-0` 保持固定高度。

### 可能原因

1. **布局问题**：画布组件使用 `absolute inset-0`，工具栏作为其子元素可能被覆盖或定位错误
2. **z-index 问题**：工具栏可能被其他元素覆盖
3. **容器高度问题**：父容器高度为 0，导致整个画布不可见
4. **CSS 样式冲突**：某些样式覆盖了工具栏的显示

### 文件分析

需要检查的文件：
- `src/components/InfiniteCanvas.tsx` - 画布组件本身
- `src/app/page.tsx` - 画布容器布局

## 修复计划

### 步骤 1：检查工具栏的 z-index 设置

当前工具栏没有设置 z-index，可能被其他元素覆盖。需要添加合适的 z-index。

### 步骤 2：检查画布容器的布局结构

确保父容器有正确的高度和定位设置，让工具栏能够正常显示。

### 步骤 3：添加调试信息

在工具栏元素上添加调试日志，确认组件是否正确渲染。

### 步骤 4：验证修复

修复后验证工具栏是否正常显示。

## 修改的文件

1. **src/components/InfiniteCanvas.tsx**
   - 为工具栏添加 z-index 设置
   - 确保工具栏样式正确

2. **src/app/page.tsx**
   - 检查画布容器的布局设置

## 风险评估

- 低风险：修改仅涉及 CSS 样式和布局，不影响核心功能逻辑
- 需要确保修改不会影响画布的其他功能（缩放、拖拽、绘制等）

## 实施步骤

1. 修改 `InfiniteCanvas.tsx` 工具栏的样式，添加 z-index
2. 检查并修复画布容器的布局问题
3. 验证类型检查通过
4. 测试工具栏是否正常显示
